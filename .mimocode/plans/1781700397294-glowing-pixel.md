# Lanhu MCP 功能增强计划

## 目标
将 lanhu-mcp 从7000+行单文件重构为多模块架构，并新增8大功能方向，让AI能获取蓝湖设计的所有细节来写页面。

---

## 一、模块化拆分（Phase 0 - 基础）

### 目标结构

### 拆分步骤
1. 创建 `lanhu_mcp/` 包目录和 `__init__.py`
2. 提取 `core/config.py`（全局配置常量）
3. 提取 `core/extractor.py`（LanhuExtractor类）
4. 提取 `core/cache.py`（缓存逻辑）
5. 提取 `core/user.py`（用户信息）
6. 提取 `converters/` 下所有转换器
7. 提取 `utils/` 下工具类（MessageStore、飞书通知等）
8. 提取 `tools/` 下所有MCP工具定义
9. 提取 `prompts/` 下所有prompt模板
10. 简化 `server.py` 为入口文件（import并注册所有工具）
11. 保持向后兼容：`lanhu_mcp_server.py` 作为兼容入口

## 二、新功能实现（Phase 1-8）

### Phase 1: 设计系统提取 `lanhu_extract_design_system`

**文件**: `tools/design_system.py`

**功能**: 从设计稿Schema JSON中提取完整设计系统规范

**输入参数**:
- `url`: 蓝湖URL
- `design_name`: 设计图名称或序号

**输出结构**:
```json
{
  "colors": {
    "primary": ["#FF730A", "#333333", ...],
    "text": [{"value": "rgba(0,0,0,0.85)", "count": 12}, ...],
    "background": [...],
    "border": [...],
    "gradient": ["linear-gradient(...)"],
    "palette": ["#色值1", "#色值2", ...]  // 去重后的完整色板
  },
  "typography": {
    "font_families": ["PingFang SC", "Roboto", ...],
    "font_specs": [
      {"size": "14px", "weight": "500", "color": "rgba(0,0,0,0.85)", "count": 8},
      ...
    ],
    "heading_styles": [...],
    "body_styles": [...]
  },
  "spacing": {
    "margin_values": [0, 8, 12, 16, 24, ...],
    "padding_values": [0, 8, 12, 16, ...],
    "gap_values": [...],
    "patterns": {"small": 8, "medium": 16, "large": 24, "xlarge": 32}
  },
  "borders": {
    "radius_values": [0, 4, 8, 12, 16, 999],
    "border_styles": ["1px solid #E5E5E5", ...]
  },
  "shadows": [
    "0 2px 8px rgba(0,0,0,0.1)",
    ...
  ],
  "components": {
    "buttons": [...],  // 按钮样式变体
    "cards": [...],     // 卡片样式变体
    "inputs": [...],    // 输入框样式
    "modals": [...]     // 弹窗样式
  },
  "design_tokens_css": ":root {\n  --color-primary: #FF730A;\n  ...}"
}
```

**实现思路**:
- 复用 `extractor.get_sketch_json()` 获取原始设计数据
- 遍历所有图层，收集：fills（颜色）、textStyle（字体）、padding/margin（间距）、radius（圆角）、shadows（阴影）
- 对相同属性值做频率统计
- 识别组件模式（通过图层命名+结构相似度）
- 生成CSS变量形式的design tokens

### Phase 2: 布局规格提取 `lanhu_get_layout_spec`

**文件**: `tools/layout_spec.py`

**功能**: 提取设计稿中每个元素的精确布局规格

**输入参数**:
- `url`: 蓝湖URL
- `design_name`: 设计图名称或序号
- `depth`: 提取深度（1=仅顶层，2=子元素，0=全部）

**输出结构**:
```json
{
  "canvas": {"width": 375, "height": 812, "device": "iPhone 13"},
  "layout_tree": {
    "name": "首页",
    "type": "container",
    "position": {"x": 0, "y": 0},
    "size": {"width": 375, "height": 812},
    "layout": "flex-column",
    "justify": "flex-start",
    "align": "stretch",
    "gap": 0,
    "children": [
      {
        "name": "导航栏",
        "type": "container",
        "position": {"x": 0, "y": 0},
        "size": {"width": 375, "height": 44},
        "layout": "flex-row",
        "justify": "space-between",
        "align": "center",
        "padding": {"top": 0, "right": 16, "bottom": 0, "left": 16},
        "children": [...]
      }
    ]
  },
  "measurements": [
    {"from": "导航栏", "to": "搜索栏", "gap": 8, "direction": "vertical"},
    {"from": "搜索栏", "to": "轮播图", "gap": 12, "direction": "vertical"}
  ]
}
```

**实现思路**:
- 从Sketch JSON提取每个图层的frame（x, y, width, height）
- 检测Flex布局特征（水平排列的子元素、均匀间距等）
- 计算相邻元素之间的间距
- 构建布局树（嵌套关系+布局类型）
- 输出结构化的布局规格

### Phase 3: 组件模式识别 `lanhu_extract_component_patterns`

**文件**: `tools/components.py`

**功能**: 自动识别设计稿中的可复用组件模式

**输入参数**:
- `url`: 蓝湖URL
- `design_names`: 设计图名称列表（支持跨设计图识别）

**输出结构**:
```json
{
  "patterns": [
    {
      "type": "button",
      "name": "主要按钮",
      "instances": 5,
      "spec": {
        "size": {"width": "auto", "height": 44},
        "padding": {"horizontal": 24, "vertical": 12},
        "border_radius": 8,
        "background": "linear-gradient(135deg, #FF730A, #FF9A00)",
        "color": "#FFFFFF",
        "font_size": 16,
        "font_weight": 600
      },
      "variants": [
        {"name": "次要按钮", "background": "transparent", "border": "1px solid #FF730A", "color": "#FF730A"}
      ]
    },
    {
      "type": "card",
      "name": "内容卡片",
      "instances": 8,
      "spec": {...}
    }
  ],
  "summary": {
    "total_patterns": 12,
    "total_instances": 56,
    "types": ["button", "card", "input", "avatar", "badge", "tag"]
  }
}
```

**实现思路**:
- 通过图层命名规则（btn_*, card_*, input_*等）+ 结构特征（相似的fills/borders/radius组合）+ 尺寸相似度进行聚类
- 对每个聚类提取统一样式规格
- 识别变体（同类型但颜色/border不同的实例）
- 生成React/Vue组件代码模板

**功能**: 从Axure原型中提取交互规格

**输入参数**:
- `url`: 蓝湖URL（必须含docId，即PRD/原型文档）
- `page_names`: 页面名称列表

**输出结构**:
```json
{
  "pages": [
    {
      "name": "登录页",
      "click_areas": [
        {
          "name": "登录按钮",
          "position": {"x": 24, "y": 480},
          "size": {"width": 327, "height": 48},
          "target": "首页",
          "action": "navigate",
          "condition": "表单验证通过"
        }
      ],
      "form_inputs": [
        {
          "name": "手机号输入框",
          "type": "text",
          "placeholder": "请输入手机号",
          "validation": "11位数字",
          "required": true
        }
      ],
      "scroll": {
        "type": "vertical",
        "content_height": 1200
      },
      "states": [
        {
          "element": "登录按钮",

          "disabled_condition": "表单未填写完整"
```

**实现思路**:

- 从Axure HTML中提取data-onclick等交互属性
- 分析表单元素的校验规则（从标注文本中提取）
- 识别页面状态变化（通过多状态组件的标注）

### Phase 5: 批量下载优化 `lanhu_batch_download_assets`

**文件**: `tools/batch_download.py`

**功能**: 一键下载设计图所有资源并按类型组织

**输入参数**:
- `url`: 蓝湖URL
- `design_name`: 设计图名称（或'all'下载全部）
- `output_dir`: 输出目录
- `scale`: 目标倍率（1x/2x/3x/ios/android）

**输出**:
```json
{
  "total_assets": 42,
  "by_type": {
    "icons": 18,
    "backgrounds": 5,
    "illustrations": 8,
    "other": 11
  },
  "manifest_path": "./assets/manifest.json",
  "directory_structure": {
    "./assets/icons/": ["icon_share.png", "icon_like.png", ...],
    "./assets/backgrounds/": ["bg_gradient.png", ...],
    "./assets/illustrations/": ["img_empty.png", ...]
  }
}
```

**实现思路**:
- 复用 `get_design_slices_info()` 获取所有切图信息
- 根据图层命名/尺寸/类型自动分类（icon < 64px、background > 200px等）
- 使用 `asyncio.gather` 并发下载
- 生成 manifest.json 索引文件
- 支持增量下载（跳过已存在的文件）

### Phase 6: 设计对比/变更追踪 `lanhu_compare_designs`

**文件**: `tools/compare.py`

**功能**: 对比同一设计图的两个版本，或同一项目的两个不同设计图

**输入参数**:
- `url`: 蓝湖URL
- `design_name`: 设计图名称
- `version_a`: 版本A（'latest'或版本号）
- `version_b`: 版本B（'previous'或版本号）

**输出结构**:
```json
{
  "summary": {
    "total_changes": 15,
    "added": 3,
    "removed": 2,
    "modified": 8,
    "layout_shifts": 2
  },
  "changes": [
    {
      "type": "modified",
      "element": "标题文字",
      "path": "导航栏/标题",
      "before": {"color": "#333333", "font_size": 16},
      "after": {"color": "#000000", "font_size": 18}
    },
    {
      "type": "added",
      "element": "新功能入口",
      "position": {"x": 24, "y": 200}
    }
  ]
}
```

**实现思路**:
- 通过 `get_sketch_json()` 获取两个版本的原始数据
- 递归遍历图层树，按name+path匹配对应元素
- 对比属性差异（颜色、大小、位置、文字内容等）
- 检测布局偏移（位置变化导致的元素重排）
- 生成变更报告

### Phase 7: 设计质量检查 `lanhu_design_qa`

**文件**: `tools/quality_check.py`

**功能**: 自动检查设计一致性、可访问性等问题

**输入参数**:
- `url`: 蓝湖URL
- `design_name`: 设计图名称

**输出结构**:
```json
{
  "score": 82,
  "issues": [
    {
      "severity": "warning",
      "category": "spacing",
      "message": "间距不一致：发现12px、13px、14px等非标准间距值",
      "elements": ["元素A", "元素B"],
      "suggestion": "建议统一使用8px倍数间距系统：8/16/24/32"
    },
    {
      "severity": "error",
      "category": "accessibility",
      "message": "文字对比度不足：#CCCCCC文字在#FFFFFF背景上对比度仅1.6:1（需≥4.5:1）",
      "elements": ["占位文字"],
      "suggestion": "建议将文字颜色调整为#999999或更深"
    },
    {
      "severity": "info",
      "category": "consistency",
      "message": "发现3种不同的圆角值：4px/5px/6px，建议统一",
      "elements": ["按钮A", "卡片B", "输入框C"]
    }
  ],
  "statistics": {
    "spacing_consistency": 0.75,
    "color_palette_size": 24,
    "font_usage_count": 5,
    "accessibility_score": 0.68
  }
}
```

**实现思路**:
- 间距检查：收集所有margin/padding值，检测非8px倍数的值
- 颜色检查：检测色板大小、颜色使用一致性、WCAG对比度
- 字体检查：检测字体族数量、字号层级是否合理（建议≤4级）
- 圆角检查：检测圆角值是否遵循统一规则
- 对比度检查：根据WCAG 2.1标准计算文字/背景对比度

### Phase 8: 框架代码生成增强 `lanhu_generate_framework_code`

**文件**: `tools/code_gen.py`

**功能**: 基于设计分析结果，生成特定框架的组件代码

**输入参数**:
- `url`: 蓝湖URL
- `design_name`: 设计图名称
- `framework`: 目标框架（react/vue/flutter/html/svelte）
- `styling`: 样式方案（css-modules/tailwind/styled-components/scss/inline）

**输出结构**:
```json
{
  "framework": "react",
  "styling": "css-modules",
  "files": [
    {
      "filename": "LoginPage.tsx",
      "content": "import styles from './LoginPage.module.css';\n..."
    },
    {
      "filename": "LoginPage.module.css",
      "content": ".container { ... }"
    }
  ],
  "assets_needed": [
    {"local": "./assets/icons/icon_share.png", "remote": "https://..."}
  ],
  "design_tokens": {
    "colors": {...},
    "spacing": {...}
  }
}
```

**实现思路**:
- 复用 `convert_lanhu_to_html()` 生成基础HTML+CSS
- 根据目标框架进行代码转译：
  - HTML+CSS → React JSX + CSS Modules
  - HTML+CSS → Vue SFC + scoped style
  - HTML+CSS → Flutter Widget
  - HTML+CSS → Svelte Component
- 保留所有CSS属性值，不做修改
- 图片路径按框架约定转换
- 生成配套的design tokens文件
